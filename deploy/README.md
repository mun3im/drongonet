# deploy/

Firmware conversion tools for deploying SEABADNet INT8 TFLite models to embedded targets.

## convert_xxd.sh

Wraps `xxd -i` to produce an `alignas(8)` C array and matching header, ready to drop into a Portenta H7 (or any bare-metal TFLM) firmware project.

```bash
bash deploy/convert_xxd.sh <model.tflite> <out_base> <symbol_prefix>
```

**Example — SEABADNet-Micro:**

```bash
bash deploy/convert_xxd.sh \
    results/6b_micro_final_fft1024_m16_s42/model.tflite \
    src/seabadnet_micro_model_data \
    g_seabadnet_model_data
```

Produces:

```
src/seabadnet_micro_model_data.h
src/seabadnet_micro_model_data.cc
```

The `.h` declares `extern const uint8_t g_seabadnet_model_data[]` and `extern const uint32_t g_seabadnet_model_data_len`. The `.cc` provides the definitions with `alignas(8)` so the array satisfies TFLM's alignment requirement.

## Target models

| Model | Script | Size (INT8) | τ | Recall |
|---|---|---|---|---|
| SEABADNet-Micro | `develop/6b_micro_final.py` | 6.56 KB | 0.35 | ≥0.98 |
| SEABADNet-Edge | `develop/6c_edge_final.py` | 33.06 KB | 0.50 (s42) | ≥0.99 |

INT8 TFLite models are generated by each training script into its result directory (`results/{name}/model.tflite`). They are not committed to the repository — regenerate from `develop/` scripts if needed.
